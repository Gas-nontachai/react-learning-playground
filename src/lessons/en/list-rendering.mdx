# 1.8 List rendering

Displaying arrays is a core React task. `Array.map` lets you turn data into JSX while the `key` prop keeps React aware of each item.

<br />

## Key rules
- Always set a stable `key` for each rendered element (id from the database beats array index)
- Keep the mapping function pureâ€”no direct mutations or async calls inside `map`
- Separate data preparation from rendering so the JSX stays simple

<br />

## Example: lessons catalog

```jsx
const lessons = [
  {slug: 'jsx', title: 'JSX basics', level: 'Beginner'},
  {slug: 'state', title: 'Thinking in state', level: 'Beginner'},
  {slug: 'use-effect', title: 'Side effects', level: 'Intermediate'}
];

function LessonList() {
  return (
    <div className="grid gap-4 md:grid-cols-3">
      {lessons.map((lesson) => (
        <article
          key={lesson.slug}
          className="rounded-2xl border border-slate-200 bg-white p-4 shadow"
        >
          <p className="text-xs uppercase text-slate-400">{lesson.level}</p>
          <h3 className="text-lg font-semibold">{lesson.title}</h3>
          <a className="text-sm text-indigo-600" href={`/lessons/${lesson.slug}`}>
            Continue â†’
          </a>
        </article>
      ))}
    </div>
  );
}
```
