# 1.7 Conditional rendering

React components often need to display different UI depending on state. Conditional rendering keeps the UI in sync with the data.

<br />

## Common patterns
- `if` statements before the return to handle early exits (e.g. loading states)
- Ternaries for inline variations: `{isOnline ? 'Online' : 'Offline'}`
- Logical AND for optional sections: `{hasError && <ErrorBanner />}`

<br />

## Example: badge selector

```jsx
import {useState} from 'react';

const badges = ['Beginner', 'Builder', 'Deployer'];

export default function BadgeSelector() {
  const [level, setLevel] = useState<string | null>(badges[0]);

  const message = level
    ? `Current badge: ${level}`
    : 'Pick a badge to continue';

  return (
    <div className="space-y-4 rounded-2xl border border-blue-200 bg-white p-6">
      <div className="flex gap-3">
        {badges.map((badge) => (
          <button
            key={badge}
            onClick={() => setLevel(level === badge ? null : badge)}
            className={[
              'rounded-full border px-3 py-1 text-sm',
              badge === level
                ? 'border-blue-600 bg-blue-600 text-white'
                : 'border-slate-300 text-slate-600'
            ].join(' ')}
          >
            {badge}
          </button>
        ))}
      </div>

      <p className="text-sm text-slate-600">{message}</p>

      {level === 'Deployer' && (
        <div className="rounded-lg border border-amber-200 bg-amber-50 p-3 text-amber-900">
```

## Tips
- Extract conditional blocks into helper components if JSX becomes hard to read
- Avoid deeply nested ternaries; prefer `if` statements to compute intermediate variables
- Always cover the â€œempty stateâ€ scenario so users never see an empty screen
