# 1.3 First component

A component is just a JavaScript function that returns JSX. Keep it small, focused, and easy to readâ€”think of it as one LEGO brick in a larger UI build.

<br />

## What makes a good component
- Has a clear purpose, e.g. `ProfileCard` renders one profile
- Receives data via props instead of reaching into global scope
- Contains only the markup and helpers it needs, nothing more

<br />

## Build a simple component

```jsx
function ProfileCard() {
  const name = "Mint";
  const role = "Frontend Student";
  const skills = ["React", "CSS", "Figma"];

  return (
    <article className="rounded-2xl border border-slate-200 bg-white p-6 shadow">
      <header className="flex items-center gap-4">
        <div className="size-14 rounded-full bg-gradient-to-br from-indigo-400 to-purple-500" />
        <div>
          <h2 className="text-xl font-semibold">{name}</h2>
          <p className="text-sm text-slate-500">{role}</p>
        </div>
      </header>
      <ul className="mt-4 flex flex-wrap gap-2 text-sm text-slate-600">
        {skills.map((skill) => (
          <li key={skill} className="rounded-full bg-slate-100 px-3 py-1">
            {skill}
          </li>
        ))}
      </ul>
    </article>
  );
}
```

This component has local variables, JSX, and array renderingâ€”exactly what you will do in real projects.

<br />

## Naming and file tips
- Use PascalCase names: `LessonList`, `Sidebar`
- Keep one component per file at the beginning; split when files grow past ~100 lines
- Export the component so other files can import it: `export default function ProfileCard() { ... }`
