# 2.4 Component Composition

Composition is the art of combining small components to form complex UIs. Rather than building a ‚Äúgod component,‚Äù break responsibilities into layers and reuse them.

<br />

## Composition patterns
- **Layout components** wrap children and handle spacing/grid concerns
- **Slot-based components** accept JSX via props (`header`, `actions`)
- **Children-as-a-function** for advanced cases needing render props

<br />

## Example: dashboard cards

```jsx
type CardProps = {
  title: string;
  description: string;
  actions?: React.ReactNode;
  children: React.ReactNode;
};

function DashboardCard({title, description, actions, children}: CardProps) {
  return (
    <section className="space-y-3 rounded-2xl border border-slate-200 bg-white p-5 shadow">
      <header className="flex items-start justify-between gap-3">
        <div>
          <h3 className="text-lg font-semibold">{title}</h3>
          <p className="text-sm text-slate-500">{description}</p>
        </div>
        {actions}
      </header>
      {children}
    </section>
  );
}

export default function Dashboard() {
  return (
    <div className="grid gap-4 md:grid-cols-2">
      <DashboardCard
        title="Students"
        description="Active learners this week"
        actions={<button className="text-sm text-indigo-600">View all</button>}
      >
        <p className="text-4xl font-bold">48</p>
      </DashboardCard>

      <DashboardCard
        title="Feedback"
        description="Latest comments"
        actions={<button className="text-sm text-slate-600">Mark read</button>}
      >
        <ul className="space-y-2 text-sm text-slate-600">
          <li>‚ÄúLoved the JSX exercise!‚Äù</li>
          <li>‚ÄúProps lesson clarified a lot.‚Äù</li>
        </ul>
      </DashboardCard>
    </div>
  );
}
```

<br />

## Tips
- Let components accept `children` so callers decide what to render inside
- Avoid prop drilling by composing context providers near the top-level layout
- Keep compositions flexible by passing render props when plain children aren‚Äôt enough
