# 1.4 Props

Props are how components receive data from their parents. They make components flexible because the same component can render different UI depending on the props you pass in.

<br />

## How props work
1. Parent renders `<Greeting name="Beam" mood="curious" />`
2. Child component receives those props as a function parameter
3. Destructure the props for clarity and optionally provide default values

<br />

## Example: Greeting component

```jsx
type GreetingProps = {
  name: string;
  mood?: 'curious' | 'focused' | 'relaxed';
};

function Greeting({name, mood = 'focused'}: GreetingProps) {
  return (
    <section className="rounded-xl border border-emerald-200 bg-emerald-50 p-4 text-emerald-900">
      <h3 className="text-lg font-semibold">Hello {name} ðŸ‘‹</h3>
      <p className="text-sm">Today's mood: {mood}</p>
    </section>
  );
}

export default function App() {
  const students = [
    {id: 1, name: 'Mint', mood: 'curious'},
    {id: 2, name: 'Beam', mood: 'relaxed'},
    {id: 3, name: 'Nam'} // falls back to default
  ];

  return (
    <div className="space-y-3">
      {students.map((student) => (
        <Greeting key={student.id} {...student} />
      ))}
    </div>
  );
}
```

<br />

## Best practices
- Use TypeScript (or PropTypes) to document expected props
- Pass only what the child needs; avoid dumping entire objects if only two fields are used
- If a prop name becomes too generic (e.g. `data` or `item`), rename it to reflect its purpose (`lesson`, `todo`, `article`)
