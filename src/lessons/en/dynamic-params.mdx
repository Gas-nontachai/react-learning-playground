## Folder syntax
- `[slug]/page.tsx` â†’ `/lessons/my-first-component`
- Multiple params: `[course]/lessons/[lesson]/page.tsx`

Next.js passes parameters to your component via the `params` prop.

<br />

## Example: lesson details

```tsx
// app/lessons/[slug]/page.tsx
import {notFound} from 'next/navigation';
import {lessonContent} from '@/lessons/registry';

const lessons = {
  'react-intro': {title: 'React intro', summary: 'Why React exists'},
  'jsx': {title: 'JSX', summary: 'Writing UI in JavaScript'}
};

type Props = {
  params: {slug: keyof typeof lessons};
};

export default function LessonPage({params}: Props) {
  const lesson = lessons[params.slug];
  if (!lesson) notFound();

  return (
    <article className="space-y-3">
      <p className="text-sm uppercase text-slate-400">Lesson</p>
      <h1 className="text-3xl font-bold">{lesson.title}</h1>
      <p className="text-slate-600">{lesson.summary}</p>
    </article>
  );
}
```

<br />

## Tips
- Validate params before using them to avoid crashes
- Use the `generateStaticParams` export to pre-render popular routes at build time
- Combine dynamic routes with `notFound()` or `redirect()` for better UX
